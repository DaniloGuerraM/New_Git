Introducción

Bienvenido al cuadernillo de trabajos prácticos sobre los principios del desarrollo web, la instalación de un servidor web NGINX y el manejo de control de versiones con Git. En este cuadernillo, exploraremos los fundamentos esenciales del desarrollo web moderno, desde la creación de sitios web estáticos hasta el despliegue de aplicaciones dinámicas en la web.

El mundo del desarrollo web es apasionante y en constante evolución. Con el crecimiento exponencial de Internet y la demanda de experiencias en línea más interactivas y sofisticadas, es fundamental para cualquier desarrollador web comprender los principios subyacentes que rigen la creación y el despliegue de sitios y aplicaciones web.

Comenzaremos explorando los conceptos básicos del desarrollo web, incluyendo los lenguajes de marcado, estilos y programación que se utilizan para construir páginas web modernas. Aprenderemos sobre HTML, CSS y JavaScript, y cómo estos lenguajes trabajan juntos para crear experiencias web ricas y atractivas.

Luego, nos sumergiremos en el mundo de los servidores web, con un enfoque en la instalación y configuración de NGINX, un servidor web ligero, rápido y altamente escalable que es ampliamente utilizado en la industria. Aprenderemos cómo configurar NGINX para servir sitios estáticos y dinámicos, y cómo gestionar y optimizar el rendimiento del servidor.

Finalmente, exploraremos el manejo de control de versiones con Git, una herramienta indispensable para cualquier desarrollador moderno. Aprenderemos los conceptos básicos de Git, incluyendo la creación de repositorios, la realización de cambios, la gestión de ramas y la colaboración en proyectos de código abierto.

Ya sea que seas un principiante absoluto en el mundo del desarrollo web o un desarrollador experimentado en busca de mejorar tus habilidades, este cuadernillo de trabajos prácticos te proporcionará los conocimientos y las habilidades necesarias para tener éxito en el emocionante y dinámico campo del desarrollo web.

Consideración Importante: Entrega Obligatoria

Este cuadernillo de trabajos prácticos es de carácter obligatorio y su entrega será la nota del primer parcial del curso. Cada ejercicio y tarea incluida en este cuadernillo contribuirá significativamente a tu comprensión de los conceptos y habilidades fundamentales necesarios para el desarrollo web, la configuración de servidores web NGINX y el manejo de control de versiones con Git.

Es importante que completes todos los trabajos prácticos de manera exhaustiva y en los plazos establecidos para garantizar tu progreso y éxito en el curso. Introducción al desarrollo web

Actividad 1: Funcionamiento Interno de los Navegadores Web

Objetivo: Investigar y comprender el funcionamiento interno de los navegadores web, incluyendo su arquitectura, motores de renderizado, gestión de memoria y seguridad.

Descripción: Investiga cómo funcionan los principales navegadores web, como Google Chrome, Mozilla Firefox y Microsoft Edge, desde el punto de vista técnico. Examina los componentes clave de un navegador web, como el motor de renderizado (como Blink o Gecko), el motor JavaScript (como V8 o SpiderMonkey), la gestión de memoria y la seguridad del navegador.

Se deberá construir un informe de no más de cuatro (4) carillas que contenga la siguiente información, incluir diagramas y texto que ayude a su comprensión.

  1. ¿Cuál es la arquitectura general de un navegador web y cuáles son sus principales componentes?
  2. ¿Qué es un motor de renderizado y cuál es su función en un navegador web?
  3. ¿Cuáles son los motores de renderizado más utilizados en los navegadores web actuales y cuáles son sus diferencias?
  4. ¿Cómo funciona el proceso de renderizado de una página web en un navegador?
  5. ¿Qué es la gestión de memoria en un navegador web y por qué es importante para su rendimiento?
  6. ¿Cuáles son los principales desafíos asociados con la gestión de memoria en los navegadores web y cómo se abordan?
  7. ¿Cómo se implementa la seguridad en un navegador web para proteger a los usuarios de amenazas como malware y ataques de phishing?
  8. ¿Cuáles son las medidas de seguridad más comunes implementadas en los navegadores web para proteger la privacidad de los usuarios y sus datos personales?
  9. ¿Qué es el sandboxing en el contexto de los navegadores web y cómo se utiliza para mejorar la seguridad?
  10. ¿Cómo se gestionan y actualizan los complementos y extensiones en un navegador web, y cuáles son los riesgos asociados con su uso?

Actividad 2: Investigación sobre Protocolos de Seguridad de Comunicaciones

Objetivo: Investigar y comprender los protocolos de seguridad de comunicaciones más importantes utilizados en Internet, como TLS/SSL, IPSec, SSH y VPN.

Descripción: Investiga los protocolos mencionados y explora cómo funcionan para garantizar la seguridad y privacidad de las comunicaciones en Internet. Examina sus características, como la autenticación, la confidencialidad, la integridad de los datos y la resistencia a los ataques. Compara y contrasta los protocolos en términos de sus aplicaciones y casos de uso específicos.

Se deberá realizar un trabajo teórico de no más de cuatro (4) carillas que contenga la siguiente información:

  1. ¿Qué es TLS/SSL y cuál es su función principal en la seguridad de las comunicaciones en Internet?
  2. ¿Cuáles son las principales características y componentes de TLS/SSL?
  3. ¿Cómo funciona el proceso de establecimiento de conexión segura utilizando TLS/SSL?
  4. ¿Cuáles son las diferencias entre TLS y SSL, y cuál es más seguro?
  5. ¿Cuál es la importancia del cifrado en TLS/SSL y cómo protege la confidencialidad de los datos transmitidos?
  6. ¿Qué son los certificados SSL/TLS y cómo se utilizan en el proceso de autenticación de servidores web?
  7. ¿Qué es IPSec y cuál es su papel en la seguridad de las comunicaciones en redes IP?
  8. ¿Cómo funciona IPSec para proporcionar autenticación, integridad y confidencialidad de los datos en la capa de red?
  9. ¿Cuáles son los modos de operación y los protocolos de seguridad utilizados en IPSec?
  10. ¿Qué es SSH (Secure Shell) y cómo se utiliza para asegurar la comunicación remota?
  11. ¿Cuál es la diferencia entre SSH1 y SSH2, y cuál es más seguro?
  12. ¿Cómo se autentican los usuarios y los servidores en una conexión SSH?
  13. ¿Qué es una VPN (Virtual Private Network) y cómo funciona para proporcionar seguridad y privacidad en las comunicaciones a través de Internet?
  14. ¿Cuáles son los diferentes tipos de VPN y cuándo se utilizan?
  15. ¿Cuáles son las ventajas y desventajas de utilizar una VPN para proteger las comunicaciones en línea?

Servidor Web

Instalación y configuración de un servidor NGINX

En esta serie de actividades prácticas, exploraremos la instalación y configuración de un servidor web NGINX, uno de los servidores web más populares y eficientes disponibles en la actualidad. NGINX es ampliamente utilizado en la industria debido a su diseño ligero, rápido y altamente escalable, lo que lo convierte en una excelente opción para alojar sitios web estáticos y dinámicos, así como para actuar como proxy inverso y equilibrador de carga.

A lo largo de estas actividades, aprenderemos cómo instalar NGINX en diferentes sistemas operativos, como Linux, Windows, y configurarlo para servir varios sitios web, gestionar redirecciones, implementar SSL/TLS para conexiones seguras, y optimizar su rendimiento para manejar cargas de tráfico elevadas de manera eficiente.

Este conjunto de actividades prácticas proporcionará a los estudiantes la oportunidad de adquirir habilidades prácticas y experiencia en la instalación y configuración de servidores web, lo que les permitirá desempeñarse de manera efectiva en roles relacionados con la administración de sistemas, el desarrollo web y la seguridad de la información.

Nota: escanear el código QR para leer un artículo sobre la instalación de NGINX en Linux y Windows.

Actividad 1: Investigación Preliminar

  • Objetivo: Familiarizarse con NGINX y su papel en la infraestructura web.
  • Descripción: Investiga sobre NGINX, su historia, características principales y casos de uso comunes en la industria. Comprende cómo NGINX se compara con otros servidores web como Apache y cuáles son sus ventajas.

Actividad 2: Instalación de NGINX

  • Objetivo: Instalar NGINX en el sistema operativo local o en un servidor de prueba.
  • Descripción: Sigue las instrucciones de instalación específicas para tu sistema operativo. Aprende a instalar NGINX desde los repositorios oficiales o mediante paquetes precompilados.

Actividad 3: Configuración del Primer Sitio Web Estático

  • Objetivo: Configurar NGINX para servir un sitio web estático.
  • Descripción: Crea un directorio para tu sitio web estático y configurar NGINX para que sirva los archivos HTML, CSS y JavaScript. Prueba el acceso al sitio web desde un navegador.

Actividad 4: Configuración de Sitios Web Virtuales (Virtual Hosts)

  • Objetivo: Configurar NGINX para servir múltiples sitios web en un solo servidor.
  • Descripción: Crea configuraciones separadas para cada sitio web en el archivo de configuración de NGINX. Aprende a asignar nombres de dominio y a dirigir el tráfico a sitios web específicos

Control de versiones

Introducción a Actividades Prácticas sobre Git y GitFlow

Bienvenido a esta serie de actividades prácticas diseñadas para enseñarte los fundamentos de Git y GitFlow, dos herramientas esenciales en el mundo del desarrollo de software colaborativo y el control de versiones.

Git es un sistema de control de versiones distribuido ampliamente utilizado que permite a los equipos de desarrollo gestionar y colaborar en proyectos de software de manera eficiente. Con Git, puedes realizar un seguimiento de los cambios en tu código, trabajar en ramas independientes, fusionar cambios y colaborar con otros desarrolladores de forma efectiva.

GitFlow, por otro lado, es un flujo de trabajo basado en Git que define una serie de reglas y prácticas para organizar y gestionar el ciclo de vida de un proyecto de software. GitFlow proporciona una estructura clara para el desarrollo de funciones, la corrección de errores y la liberación de versiones, lo que facilita la colaboración entre equipos y la entrega continua de software de alta calidad.

A lo largo de estas actividades prácticas, aprenderás cómo utilizar Git y GitFlow para gestionar proyectos de software de manera eficiente y colaborativa. Desde la creación de un repositorio Git hasta la implementación de un flujo de trabajo GitFlow completo, explorarás las principales características y funcionalidades de estas herramientas, adquiriendo habilidades prácticas que te serán útiles en tu carrera como desarrollador de software.

Actividad 1: Creación y Clonación de Repositorio en Git

  • Objetivo: Familiarizarse con los comandos básicos de Git para la creación y clonación de repositorios.
  • Descripción:
  1. Crea un nuevo repositorio vacío en GitHub.
  2. Clona el repositorio recién creado en tu máquina local utilizando el comando git clone.
  3. Crea un nuevo archivo de texto en el repositorio clonado y realiza un commit con el mensaje "Añadir archivo de prueba".
  4. Empuja los cambios al repositorio remoto en GitHub utilizando el comando git push.

Actividad 2: Implementación de GitFlow para Gestión de Ramas

  • Objetivo: Entender y practicar el uso de GitFlow para gestionar el flujo de trabajo en un proyecto.
  • Descripción:
  1. Inicia un nuevo proyecto utilizando GitFlow en un repositorio local.
  2. Crea y desarrolla una nueva característica en una rama de funcionalidad utilizando el comando git flow feature start.
  3. Fusiona la rama de funcionalidad en la rama de desarrollo utilizando el comando git flow feature finish.
  4. Crea una rama de lanzamiento para preparar una nueva versión del software utilizando el comando git flow release start.
  5. Realiza pruebas y correcciones en la rama de lanzamiento y finaliza la versión utilizando el comando git flow release finish.

Actividad 3: Colaboración y Resolución de Conflictos en Git

  • Objetivo: Practicar la colaboración en un proyecto Git y resolver conflictos de fusión.
  • Descripción:
  1. Trabaja en un proyecto colaborativo con al menos un compañero de equipo.
  2. Cada miembro del equipo debe realizar cambios en archivos diferentes en la misma rama.
  3. Intenta fusionar los cambios de tu compañero de equipo en tu rama utilizando el comando git pull.
  4. Resuelve cualquier conflicto de fusión que surja y realiza un commit con el mensaje "Resolver conflictos de fusión".
  5. Empuja tus cambios al repositorio remoto y notifica a tu compañero de equipo.
